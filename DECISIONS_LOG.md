# 动态概念库系统 - 决定记录

## ✅ 已确定的决定

### 1. 概念ID生成方案：**方案C（混合方案）**

**决定内容**：
- 使用两个ID：`id`（序列号）+ `canonical_id`（Hash）
- 文件名使用简洁的序列号：`seed_001.md`, `auto_001.md`, `manual_001.md`
- 数据库中同时存储两个ID，用于跨session一致性和去重

**数据结构**：
```json
{
  "id": "seed_001",
  "canonical_id": "seed_ai_mental_health_a1b2c3",
  "name": "AI×心理健康",
  "type": "seed",
  "created_at": "2026-04-27"
}
```

**优势**：
- ✅ 文件名简洁（seed_001.md）
- ✅ 跨session一致（canonical_id）
- ✅ 支持概念重命名（ID不变）
- ✅ 安全导入导出
- ✅ 支持三种概念来源（seed/auto/manual）
