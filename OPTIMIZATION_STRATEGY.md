# Token成本优化 - 产品经理分析

## 当前状态

### 成本结构分析

```
处理10篇论文的成本：

playmore-review：5000-7000 token（48-54%）
playmore-concepts-extract：2640-3000 token（25-23%）
playmore-notes：2000-3000 token（19-23%）

总计：9640-13000 token
成本：$0.10-0.60（取决于模型）
```

### 问题

1. **review步骤成本最高**（48-54%）
   - 但这是现有功能，用户已经在用
   - 优化空间有限

2. **concepts-extract是新增功能**（25-23%）
   - 这是我们可以优化的地方
   - 有很大的优化空间

3. **notes步骤成本中等**（19-23%）
   - 可以优化，但影响用户体验

---

## 优化方案对比

### 方案1：基础方案（当前）

**成本**：9640-13000 token

**流程**：
```
fetch → score → review → concepts-extract → notes
```

**质量**：⭐⭐⭐⭐⭐ 最高

**用户体验**：⭐⭐⭐⭐⭐ 最好

---

### 方案2：轻量级方案（激进优化）

**核心思想**：只在review中识别概念，不单独调用concepts-extract

```
fetch → score → review（包含概念识别）→ notes
```

**实现**：
```
修改playmore-review的Prompt，让Claude同时做两件事：
1. 写点评
2. 识别概念

输出：
{
  "review": "🔥必读",
  "commentary": "...",
  "concepts": [
    {
      "name": "Transformer模型",
      "confidence": 0.92
    }
  ]
}
```

**成本**：
- review：5000-7000 token（包含概念识别）
- notes：2000-3000 token
- **总计：7000-10000 token**
- **节省：25-30%**

**质量**：⭐⭐⭐⭐ 略低

**问题**：
- ❌ 概念识别精准度下降（Claude同时做两件事）
- ❌ 无法识别跨论文的概念关系
- ❌ 概念库维护困难

**用户体验**：⭐⭐⭐ 中等

---

### 方案3：智能缓存方案（推荐）

**核心思想**：缓存已识别的概念，避免重复识别

```
第一次搜索：
  fetch → score → review → concepts-extract → notes
  成本：9640-13000 token

第二次搜索（相同或相似关键词）：
  fetch → score → review → concepts-extract（使用缓存）→ notes
  成本：7000-9000 token（节省30-40%）

第三次及以后：
  成本继续降低
```

**实现**：
```python
# 缓存结构
concept_cache = {
    "canonical_id": {
        "name": "Transformer模型",
        "keywords": ["transformer", "attention"],
        "confidence": 0.92,
        "cached_at": "2026-04-27",
        "hit_count": 5
    }
}

# 查询逻辑
def extract_concepts(papers):
    new_concepts = []
    cached_concepts = []
    
    for paper in papers:
        # 检查是否在缓存中
        for concept_id, concept in concept_cache.items():
            if concept_matches(paper, concept):
                cached_concepts.append(concept)
            else:
                new_concepts.append(paper)
    
    # 只对新概念调用Claude
    if new_concepts:
        identified = call_claude_concepts(new_concepts)
        # 更新缓存
        for concept in identified:
            concept_cache[concept["canonical_id"]] = concept
    
    return cached_concepts + identified
```

**成本**：
- 第1次搜索：9640-13000 token
- 第2-5次搜索：7000-9000 token（平均节省35%）
- 长期平均：7500-10000 token

**质量**：⭐⭐⭐⭐⭐ 最高

**用户体验**：⭐⭐⭐⭐⭐ 最好

**优点**：
- ✅ 保证质量
- ✅ 随着使用越来越便宜
- ✅ 用户体验最好
- ✅ 易于实现

### 方案4：分层方案（平衡）

**核心思想**：根据用户需求提供不同的处理深度

```
用户选择1：快速模式（省钱）
  fetch → score → review（简化）
  成本：3000-4000 token
  质量：⭐⭐⭐

用户选择2：标准模式（推荐）
  fetch → score → review → concepts-extract → notes
  成本：9640-13000 token
  质量：⭐⭐⭐⭐⭐

用户选择3：深度模式（最全）
  fetch → score → review → concepts-extract → concepts-organize → notes
  成本：12000-16000 token
  质量：⭐⭐⭐⭐⭐⭐
```

**实现**：
```bash
# 快速模式
/playmore --fast AI 心理健康

# 标准模式（默认）
/playmore AI 心理健康

# 深度模式
/playmore --deep AI 心理健康
```

**成本**：
- 快速：3000-4000 token
- 标准：9640-13000 token
- 深度：12000-16000 token

**质量**：
- 快速：⭐⭐⭐
- 标准：⭐⭐⭐⭐⭐
- 深度：⭐⭐⭐⭐⭐⭐

**用户体验**：⭐⭐⭐⭐ 好

**优点**：
- ✅ 用户可以选择
- ✅ 满足不同需求
- ✅ 成本灵活

**缺点**：
- ⚠️ 增加复杂度
- ⚠️ 用户需要理解不同模式

---

### 方案5：混合方案（最优）

**核心思想**：结合方案3（缓存）+ 方案4（分层）

```
快速模式 + 缓存：
  成本：2000-3000 token（节省70%）
  质量：⭐⭐⭐

标准模式 + 缓存：
  成本：7000-10000 token（节省25-35%）
  质量：⭐⭐⭐⭐⭐

深度模式 + 缓存：
  成本：10000-13000 token（节省15-20%）
  质量：⭐⭐⭐⭐⭐⭐
```

**实现**：
```bash
# 快速模式（首次）
/playmore --fast AI 心理健康

# 快速模式（后续）
/playmore --fast AI 心理健康

```

**成本**：
- 长期平均：5000-8000 token（节省40-50%）

**质量**：⭐⭐⭐⭐⭐ 保持最高

**用户体验**：⭐⭐⭐⭐⭐ 最好
